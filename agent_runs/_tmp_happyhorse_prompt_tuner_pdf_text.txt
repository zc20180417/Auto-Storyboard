三、 Skill

1.

四、 Skill
 如何解决实际痛点
痛点⼀： Prompt
 很⻓，但不知道该删什么
Skill 会先区分两类信息：
可删信息：空泛⻛格词、重复强调、⽆效技术词、负⾯提示、与画⾯⽆关的形容。
不可删信息：关键镜头、动作阶段、产品展示步骤、⻆⾊关系、台词、声⾳设计、参考图
锚点。
这能避免把 “ 压缩  Prompt” 误做成 “ 删掉需求 ” 。
痛点⼆：需求很复杂，但不能直接放弃
例如打⽃、跑酷、赛⻋、爆炸、机甲变身这类需求， HH 可能不擅⻓完整连续⾼动态表现，
但业务上⼜不能直接取消。 Skill 的处理⽅式是：
1. 先保留⽤户真正要看的戏核。
2. 把连续⾼动态拆成更清楚的动作触点。
3. ⽤镜头、光影、尘⼟、⽔雾、碎⽚、⾐料反应补强张⼒。
4. 只有在⼈⼯确认后，才把完整动作降级为单⼀关键瞬间。
痛点三： I2V
 容易偏离⾸帧
I2V 的  Prompt 不能只写动作，还要把⾸帧当作事实基础。 Skill 会检查：
主体当前朝向是什么？
镜头在正⾯、侧⾯、低⻆度还是俯视？
物体当前已经处于什么物理状态？
动作应该从画⾯内哪个位置继续？
是否因为改写导致赛⻋从正⾯跳到背⾯、⼈物换了机位、物体关系被重置？
痛点四： R2V
 容易丢失⼈物和产品意图
R2V 的  Prompt 既要调⽤参考图，⼜要说明参考图之间的关系。 Skill 会要求：
character1 / character2 分别是谁。
哪些参考图是⼈物身份，哪些是产品，哪些是场景。
⼈物和产品在镜头中的相对位置。
哪些台词和产品展示步骤必须保留。
复杂  Prompt 难理解 直接⼤幅压缩 区分低价值废话和⾼价值镜
