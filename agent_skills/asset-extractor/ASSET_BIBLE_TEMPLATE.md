# 《剧名》全局资产库

本文件用于锁定多集资产抽取的一致性。分集 worker 只能读取本文件，不要并发改写；新增基础资产或状态先写入分集 `assets.md`、`asset_status.json` 的 `bible_update_candidates` 或交付说明，最后由主线程统一合并。

## 一、基础人物资产

基础人物资产只记录不会轻易变化的身份和外观锚点，不写某一集的表情、动作、台词状态。

| asset_id | 角色名 | 身份/关系 | 年龄段/性别 | 身高体态 | 面部稳定特征 | 发型 | 气质关键词 | 基础提示词 |
|---|---|---|---|---|---|---|---|---|
| CHAR_角色名_BASE |  |  |  |  |  |  |  |  |

## 二、人物状态资产

人物状态用于记录服装变化、脏污、湿身、受伤、年龄阶段、身份阶段等会影响多镜头一致性的变化。短暂表情、眼神、手势通常不入库。

| state_id | asset_id | 状态类型 | 状态名称 | 服装ID | 外观变化 | 首次出现 | 复用范围 | 状态提示词 |
|---|---|---|---|---|---|---|---|---|
| STATE_角色名_状态_V1 | CHAR_角色名_BASE |  |  | COSTUME_角色名_服装_V1 |  | ep01 | global / episode_range / one_episode |  |

## 三、服装资产

服装资产是人物状态中服装细节的权威来源。人物状态可以引用服装ID，不要自由改写另一套颜色、款式或鞋配。

| costume_id | 对应角色asset_id | 服装名称 | 描述 | 面料/颜色 | 年代感/磨损 | 复用范围 | 提示词 |
|---|---|---|---|---|---|---|---|
| COSTUME_角色名_服装_V1 | CHAR_角色名_BASE |  |  |  |  | global / episode_range |  |

## 四、场景基础资产

场景基础资产必须是空镜，记录空间结构、主要材质、核心陈设和基础光线，不写人物、人脸、人群或背影。

| scene_id | 场景名称 | 空间结构 | 主要材质 | 核心陈设 | 基础光线 | 基础提示词 |
|---|---|---|---|---|---|---|
| SCENE_地点_BASE |  |  |  |  |  |  |

## 五、场景状态资产

场景状态用于记录同一场景的白天、夜雨、断电、爆炸后、整洁后、破损后等可复用变化。

| scene_state_id | scene_id | 状态类型 | 光线/时间 | 空间状态 | 变化字段 | 首次出现 | 状态提示词 |
|---|---|---|---|---|---|---|---|
| STATE_SCENE_地点_状态_V1 | SCENE_地点_BASE | lighting_time / scene_condition |  |  |  | ep01 |  |

## 七、道具状态资产

道具状态用于记录完整、破损、盖章、沾血、写字据后、现金散落等影响连续性的变化。道具文字只能来自分镜或剧本明确内容。

| prop_state_id | prop_id | 状态类型 | 状态说明 | 变化字段 | 首次出现 | 状态提示词 |
|---|---|---|---|---|---|---|

## 八、特殊构图资产

特殊构图资产用于稳定反复复用的构图、透视、遮挡和空间关系。可以包含人物关系，但人物脸、发型和服装必须引用人物/服装资产，不得在这里新增设定。

| composition_id | 构图名称 | 关联资产ID | 是否含人物 | 构图用途 | 复用范围 | 提示词 |
|---|---|---|---|---|---|---|
| COMPOSITION_构图名_用途_V1 |  |  | 是/否 |  | global / episode_range / one_episode |  |

## 九、项目统一风格

| 项目 | 内容 |
|---|---|
| 画幅 | 竖屏构图 |
| 质感 | 电影质感，真实光影，真实材质，避免卡通、3D 渲染、塑料感 |
| 人物 | 保持中国短剧人物审美，不使用西方人脸 |
| 场景 | 场景资产必须为空镜、无人、无人脸 |
| 负面词策略 | 正向提示词和负面提示词分列维护；目标模型支持 `--neg` 时才内联 |
| 年代/题材关键词 |  |
| 禁用新增元素 |  |
