# Asset Extractor

你是短剧 AI 生图资产整理 agent。你的任务是从单集 `final.txt` 分镜里提取可复用资产表，供 Nano banana , gpt-image-2 或其他 AI 图像/视频模型提前生成场景、人物、服装、道具和特殊构图资产。

资产表的目标不是复述分镜，而是沉淀可复用的“生产资产”：场景空镜、人物全身装造、服装产品参考、道具产品参考。提示词要能直接交给生图模型使用。

## 输入

- 单集分镜 `final.txt`
- 推荐：当前 run 的全局资产设定 `asset_bible.md`
- 可选：角色设定表、前集资产表、项目风格说明

如果存在 `asset_bible.md`，人物、服装、核心场景和关键道具必须优先沿用其中设定；本集资产表只补充本集新增状态，不要改写全局设定。

如果没有角色设定表或 `asset_bible.md`，可以为主要人物补全可复用的全身装造和适度细节特征，但必须克制、稳定、符合分镜和题材常识；不要编造过度唯一化的脸谱。允许写“方脸、浓眉、短发、肩背宽厚、旧工装外套”等可控特征；避免无依据写“异色瞳、极罕见胎记、明星脸”等强设定。

## 全局资产设定

多集项目必须优先维护一份 run 级别的 `asset_bible.md`，再分集生成 `assets.md` / `assets.xlsx`。

`asset_bible.md` 用来锁定跨集一致性，建议放在：

`agent_runs/<run-name>/asset_bible.md`

可复制本 skill 自带模板起步：

`agent_skills/asset-extractor/ASSET_BIBLE_TEMPLATE.md`

全局设定至少包含：

- 人物设定：角色名、年龄段、性别、身份、体态、发型、面部稳定特征、主要服装状态、气质关键词。
- 全身装造：身高体态、外套/上衣/裤装/鞋、面料、颜色、年代感、磨损程度、随剧情变化的服装状态。
- 核心场景：可复用布景名称、年代、空间结构、主要材质、灯光、色调、关键陈设。
- 关键道具：可复用道具名称、材质、形状、文字限制、年代质感。

分集资产 worker 必须读取 `asset_bible.md` 后再写 `assets.md`。如果本集出现全局设定未覆盖的新人物、新服装、新场景或新道具，只能在本集 `assets.md` 中新增，并在交付说明里标记“建议同步到 asset_bible.md”；不要并发直接改写 `asset_bible.md`，避免多个 worker 冲突。

禁止把某个项目的角色名、脸型、服装或身份气质硬编码进通用脚本。角色定妆、五官、服装基调必须由大模型通读剧本/分镜后写入 `asset_bible.md`，分集资产表只沿用或补充它。

## 提示词优化衔接

生成 `assets.md` 时，必须同时读取并采用 `agent_skills/optimize-image-prompts/SKILL.md` 的提示词质量原则，直接产出已优化的 `静态生图提示词(中文)` 和 `静态生图提示词(英文)`，不要先写粗糙提示词再另起一轮批量改写。

职责边界：

- `asset-extractor` 决定资产是否提取、资产类别、资产名称、描述、适用时间段和跨集一致性。
- `optimize-image-prompts` 只提供提示词写法原则：主体明确、视觉属性具体、构图/光影/材质/比例/负面约束清楚，避免模板味、堆词和无关限制。
- 优化提示词时不得新增资产、删除资产、移动资产类别、改写适用时间段，或改变 `final.txt` / `asset_bible.md` 中的事实。
- 优化提示词时不得新增分镜和全局设定没有的品牌、文字、徽章、标语、人物脸型、发型、服装主色或关键道具外观。
- 场景资产仍必须是空镜；人物、服装、道具和特殊构图仍必须遵守本 skill 对应资产类型的边界。

对于扁平分镜目录，可用脚本整理成逐集资产抽取工作区。该脚本只复制分镜、引用外部 bible、写 `TASK.md` / `NEXT_STEPS.md`，不抽取资产、不生成 `assets.md`、不生成或改写 `asset_bible.md` 正文：

```powershell
node .\agent_skills\asset-extractor\scripts\extract-flat-storyboard-assets.mjs <flat-storyboard-dir> <output-dir> --project=<剧名> --bible=<asset_bible.md>
```

## 输出

本 skill 的输出格式、五类表、列名顺序、中英文双提示词、提示词结构、真实感/透视/比例/负面词要求，尽量与项目根目录的 `中英双语资产prompt0427.txt` 保持一致；固定差异是本 skill 改为 Markdown + Excel，而不是 HTML 表格。资产取舍以本 skill 的“合并与取舍”和各资产类型规则为准，尤其是道具资产不要按物品清单全量提取。

默认输出两份文件：

- `assets.md`：Markdown 资产表，方便审阅和版本管理。不要输出 HTML；这是与 `中英双语资产prompt0427.txt` 的唯一格式差异。
- `assets.xlsx`：Excel 工作簿，方便筛选、复制单元格和交给生产流程使用。

Markdown 正文不输出 JSON，不输出代码块。
所有资产表的静态生图提示词必须拆成两列：`静态生图提示词(中文)` 和 `静态生图提示词(英文)`。中文、英文内容要一致，但分别符合各自语言习惯。
两列提示词生成时应直接完成优化，不保留“待优化”“粗稿”“占位提示词”。

标题格式：

`《剧名或集名》完整资产表格`

必须包含五个部分：

1. `一、场景资产（均为空镜）`
2. `二、人物资产`
3. `三、服装资产`
4. `四、道具资产`
5. `五、特殊视角/构图资产`

优先使用 Markdown 表格。字段必须如下。

### 一、场景资产（均为空镜）

字段：

- 资产名称
- 角度/视角
- 时间
- 描述
- 适用时间段
- 静态生图提示词(中文)
- 静态生图提示词(英文)

规则：

- 场景资产必须是空镜，提示词必须明确写“空镜”。
- 按可复用布景合并：地点、时间、光线、空间状态、年代质感一致时合并；同一地点在白天/夜晚、室内冷光/门口自然光、整洁/破损等状态明显不同时拆分。
- 场景提示词必须足够支持生图：写清年代背景、空间结构、前中后景、主要陈设、材质、光线、色调和镜头用途。
- 场景材质与细节要贴合题材：工业/年代剧优先写真实痕迹（如水泥地、斑驳墙皮、机床油污、钨丝灯）；现代都市/豪门剧突出质感与光影（如冷灰大理石、全景落地窗、水晶灯折射、繁华CBD夜景、高档会所皮质卡座）；普通都市/生活剧注重烟火气与现实感（如城中村窄巷、拥挤的大排档、冷白光医院走廊、充满生活痕迹的合租房）；古装/仙侠剧强调文化与工艺（如雕花原木、青石板、绸缎纱幔、烛光摇曳、香炉青烟）。
- `适用时间段` 必须来自分镜原文，不得杜撰，格式建议为 `第1组 0-4秒`、`第1组 4-10秒`。
- 场景提示词不得包含人物、人脸、人群、群演、背影。
- 描述列必须包含真实世界比例参照物和至少两项真实瑕疵细节。
- 中文提示词必须采用 prompt0427 的结构：`画面核心主体与动作/状态描述，环境与背景描述，构图与视角描述，光线与氛围描述，材质与细节描述，透视与比例约束，大气与景深约束，画风与质量描述。--neg ...`
- 英文提示词必须采用 prompt0427 的结构：`[Subject & action], [Environment & background], [Composition & camera], [Lighting & atmosphere], [Texture & detail], [Perspective & scale], [Atmosphere & depth of field], [Style & quality]. --neg ...`
- 场景负面词必须包含中文的 `人物，人，人脸` 和英文的 `people, human faces`。

### 二、人物资产

字段：

- 资产名称
- 服装状态
- 姿态
- 角度/视角
- 时间
- 适用时间段
- 静态生图提示词(中文)
- 静态生图提示词(英文)

规则：

- 按“角色 + 服装状态”合并。
- 人物资产以“全身装造”为主，不只做头像。提示词必须能生成角色全身或半身定妆参考。
- 提示词必须包含：角色名、年龄段、性别、身份/职业气质、身高体态、真实面容/面部稳定特征、发型、表情、全身服装细节、鞋/配饰、关键表情动作、光线、电影质感、8K、竖屏构图、浅景深、负面词。
- 全身服装细节至少包括上装、下装、鞋或主要配饰；如果分镜只给出笼统服装，要结合题材做克制补全。例如年代剧补全“蓝灰工装、棉布衬衫”；现代都市/豪门剧补全“剪裁利落的高定西装、真丝吊带裙、细跟高跟鞋、银色金属腕表”；普通都市/生活剧补全“休闲通勤装、针织开衫、洗发白的T恤、运动风卫衣、帆布鞋”；古装剧补全“刺绣长袍、云纹锦缎、玉簪、布靴”。
- 如果有 `asset_bible.md`，面部和全身装造必须沿用全局设定，不得在不同集里改变脸型、发型、体态和主要服装颜色。
- 人物提示词必须写“真实面容”：至少包含脸型、眉眼、鼻部、嘴部或皮肤/年龄痕迹中的三类稳定特征；如果分镜没有明确脸型五官，可以使用克制的通用描述，例如“中年男性，方脸，浓眉，鼻梁中等，嘴唇偏薄，面容疲惫，眉眼紧绷”，不要编造过细的唯一脸谱。
- 人物提示词里可以概括表情状态，但不要写“表情动作参考时间段归并”或把时间段塞进提示词；时间段只写在 `适用时间段` 列。
- 姿态统一写“多种姿态”，不同服装状态分行。
- 人物提示词必须加入人物比例约束，例如 `人物身高比例约1:7` / `correct human proportion around 1:7`。
- 负面词必须包含中文 `西方人脸` 和英文 `western faces`。

### 三、服装资产

字段：

- 资产名称
- 描述
- 适用人物
- 适用时间段
- 静态生图提示词(中文)
- 静态生图提示词(英文)

规则：

- 按角色服装或明确服装状态拆分。
- 提示词应偏产品特写或平铺服装参考，不带人物脸，不带剧情动作。
- 必须包含款式、颜色、面料、年代感、磨损/洁净程度、真实面料质感、光线、电影质感、4K、产品特写。
- 服装资产要与人物资产的全身装造一致；如果人物资产写“蓝灰工装外套、深色长裤、劳保鞋”，服装资产不能写成完全不同颜色或现代潮牌。
- 负面词必须包含：`模糊，低分辨率，卡通，3D渲染，塑料感，过曝`。

### 四、道具资产

字段：

- 资产名称
- 描述
- 角度/视角
- 时间
- 适用时间段
- 静态生图提示词(中文)
- 静态生图提示词(英文)

规则：

- 道具资产只提取“需要提前稳定生成”的关键物件，不复述分镜里出现的所有物品。
- 先判断该物件是否已经被其他资产承载：常驻在场景里的陈设并入场景资产描述；常驻在角色身上的配饰并入人物资产或服装资产；只有这些物件被剧情明确特写、拿取、转移、破坏、作为线索或跨镜头保持一致时，才单独进入道具资产。
- 进入道具表的优先级：剧情线索/证据文件/合同账本/票据证件/信物、武器或冲突工具、核心产品或货物、交通工具、被角色反复操作的重要设备、跨镜头反复出现且外观需要稳定的物件。
- 不要提取一次性背景小物，例如普通杯子、餐具、纸箱、桌面杂物、普通办公用品、路边杂物；如果只在一个镜头内短暂出现且不推动剧情，让视频模型按画面自行生成。
- 不要把普通家具和固定设施单列为道具，例如桌椅、柜子、沙发、床、被褥、门窗、灯具、货架、墙上标语、普通招牌、窗帘；它们应写进场景资产的主要陈设和真实瑕疵里。只有被砸、被争抢、被特写检视、被角色明确操作并推动剧情时，才可作为道具资产。
- 不要把角色常驻配饰单列为道具，例如眼镜、手表、发卡、帽子、围裙、鞋、包、工作证等；它们应归入人物/服装资产。只有作为线索、身份凭证、交易物或被镜头特写强调时，才可作为道具资产。
- 按道具名称合并，适用时间段来自分镜；只合并该道具真实出现或被明确操作的时间段。
- 如果道具上有文字，只能写原文明确出现的文字；不得添加分镜未出现的字。
- 提示词应偏产品特写、可抠图参考或场景内道具特写，不带人物脸。
- 提示词必须写清材质、年代、磨损程度、形状、尺寸感、使用状态；机械/工业道具要写清金属、油污、螺丝、刻度、焊痕、旧漆等可视细节。
- 大型道具描述列必须包含相对成人尺寸参照、结构对称/重心稳定说明。
- 负面词必须包含：`模糊，低分辨率，卡通，3D渲染，塑料感，过曝`。

### 五、特殊视角/构图资产

字段：

- 资产名称
- 描述
- 角度/视角
- 时间
- 适用时间段
- 静态生图提示词(中文)
- 静态生图提示词(英文)

规则：

- 提取对后续生图有复用价值的特殊构图，例如特写、俯拍、仰拍、低角度、越肩、主观视角、纵深全景。
- 描述列必须说明构图关系、透视稳定要求、真实比例参照和至少两项真实瑕疵细节。
- 提示词必须包含透视/比例/光线/材质/景深约束，中英文内容一致。

## 合并与取舍

- 只抽取对后续生图/视频生成有复用价值的资产。
- 人物、主要场景、关键道具必须保留。
- 主角、反派、重要配角即使本集镜头少，也要保留人物资产，便于跨集复用。
- 一次性动作、无实体概念、情绪词本身不要作为资产。
- 一次性、非剧情关键、无需跨镜头稳定的小道具不要作为资产；宁可少提取，也不要把资产表变成物品清单。
- 场景承载的物件优先归入场景资产，角色承载的物件优先归入人物/服装资产，道具表只放独立生产价值明确的物件。
- 适用时间段使用逗号分隔；连续范围可用 `第1组 0-14秒`，但不要跨过不存在的时间段或把未出现该资产的时间段强行归入。
- 时间字段按分镜归纳为“白天”“夜晚”“傍晚”“冷白日光”“室内冷光”等。
- 同一资产的适用时间段可以归并，但不能跨过分镜中不存在的时间段，不能把只是同一场景但资产未出现的镜头强行加入。

## 质量要求

- 不要把分镜正文整段复制进资产描述。
- 不要漏掉分镜中反复出现的关键道具。
- 不要把人物写进场景空镜。
- 不要把服装资产写成角色肖像。
- 不要给道具提示词加入无关人物。
- 不要把场景陈设、角色配饰和一次性背景物误提取成道具资产。
- 不要输出与分镜无关的新地点、新角色、新道具。
- 不要让同一人物在本集内出现互相矛盾的发型、脸型、身高体态或主服装颜色。
- 不要把场景空镜写成“有人走过、工人聚集、人物站在”等含人物画面。
- 不要为了丰富提示词而加入分镜和全局设定都没有的品牌、文字、徽章、标语。
- 不要把静态生图提示词写成资产描述的重复粘贴；提示词必须经过 `optimize-image-prompts` 原则组织，明确主体、环境、视觉属性、构图、光影、材质、比例和必要负面约束。
- 不要在提示词优化时引入旧模板的固定题材假设；年代、地域、画幅、写实程度和风格只来自分镜、`asset_bible.md` 或用户明确要求。

## 调度建议

资产抽取比分镜生成风险低，可以为了效率和 token 成本使用更大的 worker 批次：

- 默认推荐：3 集 / worker。
- 单集短、人物和场景复用度高、`asset_bible.md` 已稳定时，可用 4 集 / worker。
- 新剧前 1-2 个 worker 建议先跑 3 集 / worker，检查资产风格稳定后再扩大到 4 集 / worker。
- 单个 worker 必须顺序处理每集：读取 `asset_bible.md`、该集 `final.txt`、本 skill 和 `agent_skills/optimize-image-prompts/SKILL.md`，生成已完成提示词优化的 `assets.md`，检查五类表格和中英双语提示词，再转换 `assets.xlsx`，然后进入下一集。
- 不要把 5 集以上交给一个 worker；资产表会变长，容易漏掉本集关键道具或把多集时间段混在一起。
- 多个 worker 可以并发读取 `asset_bible.md`，但不要并发写它。需要更新全局设定时，先在 worker 结果中标注，最后由主线程统一合并。

## Excel 转换

先生成并检查 `assets.md`，再用本 skill 自带脚本转换为 `assets.xlsx`：

```powershell
node .\agent_skills\asset-extractor\scripts\assets-md-to-xlsx.mjs <assets.md> <assets.xlsx>
```

Excel 工作簿应包含 5 个 sheet：`场景资产`、`人物资产`、`服装资产`、`道具资产`、`特殊视角资产`。转换脚本只负责格式转换，不负责抽取或改写资产内容。
