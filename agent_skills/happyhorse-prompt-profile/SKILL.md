# HappyHorse 1.0 Prompt Profile

本文根据《HappyHorse 1.0 系列模型使用指南》整理，用作 Auto-Storyboard 的内部参考层。

它不替代 `storyboard-generator` 的剧情忠实、时长、`cut_id`、reviewer 和 validate gate 规则；但当任务明确指定 HappyHorse 目标模型时，它定义本集 `final.txt` 的组内提示词外壳。

## 关键边界

Auto-Storyboard 的生视频流程默认会在生产侧关联多张参考图，用于固定人物、服装、场景、道具和构图资产。这个能力对 Seedance 2.0 和 HappyHorse 1.0 都成立。

因此，分镜正文只写视频片段提示词本身，不写参考图、首帧、尾帧、`@图片1`、`Image 1`、`参考图1` 等工程占位，也不解释参考图如何绑定。参考图关联由资产绑定和生产界面处理。

HappyHorse 与 Seedance 的差异不在“有没有参考图”，而在同样使用多参考图 + 提示词生产视频时，HappyHorse 更偏好的文字组织方式。

## 与 ai-video-prompt 的关系

`agent_skills/ai-video-prompt/SKILL.md` 只在准备工作区显式指定 HappyHorse 目标模型时作为补充参考；默认 Seedance 分镜不得读取或套用它。

可以借鉴的只有提示词表达技巧：`场景 / 主体 / 运动 / 音频` 的集中组织、正向排除、视觉代偿、具体动作替换抽象词、稳定镜头控制、声音来源和台词语气说明。

必须过滤掉通用视频平台写法：`@图`、`Image 1`、参考图槽位、首帧/尾帧、视频编辑、BGM、独立音频时间轴、`[0-3s]` 时间标记、`s` 单位、`[]` / `<>` 声学控制原始标记。需要声音信息时，改写成 `【音频】` 里的自然中文，并用“第一段/第二段/第三段”对应 `【运动】` 段落。

## 使用原则

- HappyHorse prompt 的核心组织是：场景 + 主体 + 运动 + 音频。
- 每组仍是一次独立生成的视频片段，组首必须锁定空间、人物位置、道具归属和视线关系。
- 短剧分镜优先使用中景、近景、对峙、反应镜头和清晰口型承载；不要为了炫技牺牲台词、人物关系和剧情因果。
- 静态外观和跨集一致性由资产图承载，分镜时间段只承载动作、台词、反应、视线、道具操作、声音和必要运镜。
- 分镜正文不要写模型说明词，例如“HappyHorse 可自动分镜”“模型会识别参考图”“由模型补全”等。
- 官方示例中的广告、产品展示、萌宠、MV、风格转换、视频编辑和一镜到底写法，只能在用户明确要求对应场景时参考；默认短剧流程不吸收这些语气。

## Prompt Tuner 原则

HappyHorse 目标下，分镜 worker 的默认调优深度是“最小修补 / 结构清理”，不是激进重写。除非用户明确要求重写，不得为了稳定画面而删除关键剧情、关键道具、关键动作阶段、原剧本台词、反转点或声音设计。

每组生成前先做内部检查，不写进 `final.txt`：

1. 必须保留：本组关键镜头、动作阶段、人物关系、关键道具、原剧本台词、画外音/心声/环境声。
2. 可压缩：重复风格词、泛泛情绪词、长串负向词、生产界面参数、与画面无关的形容。
3. 五维补锚点：主体、场景、运动、镜头、声音。判断哪些信息已由剧本或资产图锁定，只补空缺维度。
4. HH 风险处理：复杂动作不直接删，先拆成动作触点、身体方向、受力反馈、衣料/尘土/水雾/道具反应。
5. 正向表达优先：少写“不要变形/不要穿帮”，多写“手指收紧、肩膀后退半步、尘土被脚步带起、金属边缘反光”。

## HappyHorse 分镜写法

每组检查这五类信息是否在自然分镜中成立：

1. 场景：写清物理空间、时间、光线、主要陈设和气氛，但不要把空间介绍单独占成一个慢镜头。
2. 主体：写清本组可见人物、画面位置、身体朝向、对话对象和道具归属。
3. 运动：写连续动作链、视线变化、表情变化、道具操作和镜头运动；动作必须可拍、可演、可同步。
4. 镜头：写清景别、机位、稳定运镜和前中后景层次；优先固定、微推、侧推、过肩、低角度等可控动作。
5. 声音：写清现场对白、画外音、电话音、门外音、环境声；现场对白默认有口型，心声和画外音要写明嘴唇闭合、不做口型。

HappyHorse 适合表达“电影质感、动作自然、镜头稳定、转场自然、面部表情细腻”。这些词可以进入画面风格或镜头描述，但不能替代具体画面。

## 推荐组内结构

HappyHorse 目标模型下，每组必须使用“场景 / 主体 / 运动 / 音频 / 画面风格”的专属外壳，并嵌入 Auto-Storyboard 现有组格式，保留 `cut_id`、组标题、中文 `秒` 时间段、组尾衔接和可校验时长。

不要继续使用默认 Seedance 外观：不要单独写 `**人物**`、`**场景**`、`**道具**`、`组首空间锁定`、`画面风格：`、独立 `--neg` 行。人物、道具、空间锁定信息都写进 `【场景】` 和 `【主体】`；画面风格和负向词都写进 `【画面风格】`。

推荐写法：

```text
=== [cut_id: EPxx-GNN] 第N组：组名（总时长：XX秒，镜头数：X个） ===

【场景】写本组物理空间、时间、光线、主要陈设和空间氛围。不要写参考图、首帧或资产绑定说明。
【主体】人物：写本组可见人物列表；关键道具：写本组关键道具列表；主体关系：写人物位置、身体朝向、对话对象、关键道具归属和潜伏/遮挡关系。

【运动】
0-4.5秒：
镜头描述：写镜头景别、机位、人物动作链、表情变化、道具操作和必要运镜；不要在这里写完整台词，只写开口动作、口型对象和可见反应。
光影设计：写本时间段的光线方向、色调和面部可见性。

4.5-11秒：
镜头描述：继续写下一段动作、反应和镜头运动；台词正文放到【音频】。
光影设计：写光线延续或变化。

【音频】与【运动】段落同步写声音信息，但不要重复时间码。第一段：写环境声、动作声、现场对白或画外音来源；现场对白写清谁对谁说，并保留原剧本台词。第二段：继续写对白、脚步、刹车、风声、虫鸣等声音。
组尾衔接：该组以[人物/道具/空间状态]自然收尾。不强制静止。
【画面风格】真人实拍风格，电影质感，浅景深，高清细节，动作自然流畅，无字幕，无配乐。负向：字幕，水印，logo，低清，畸变，卡通感，面部融合，文字叠加。

=== 第N组结束 ===
```

注意：

- 时间段只能在【运动】里作为独立行出现一次，例如 `0-4.5秒：`。不要在【音频】里再单独写一套 `0-4.5秒：`，否则机械校验会把它当成额外镜头。
- 使用 `【音频】` 承载 HappyHorse 需要的声音信息；用“第一段/第二段/第三段”对应【运动】里的时间段，不写第二套时间轴。
- 不使用 `s` 作为单位，统一写中文 `秒`。
- `【画面风格】` 必须吸收原来的“画面风格”和负向词；HappyHorse 目标下不要单独输出 `画面风格：` 或 `--neg`。
- 不要在 `【画面风格】` 里写生产界面参数，例如 `16:9`、`9:16`、`11秒`、`11s`、`4K`、`1080p`。比例、时长、分辨率由生产界面和组标题承担，不进入提示词正文。
- `【画面风格】` 应短，不要每组重复长负向词。负向词如需写，最多 8 项，优先写字幕、水印、logo、低清、畸变、卡通感、面部融合、文字叠加。
- 原剧本台词只在 `【音频】` 里完整出现；`【运动】` 只写“开口说话、口型清楚、看向谁、身体反应”等可见动作。
- `character1`、`character2`、`character3` 这类占位名如果来自上游资产绑定，可以保留；不要改写成未授权的新角色名。

## HappyHorse 友好表达

短剧常用表达可以优先采用：

- “真人实拍风格，电影级光影质感，面部细节清晰，动作自然流畅。”
- “固定机位”“镜头微推”“缓慢侧推”“平视中景”“过肩近景”“低角度压迫构图”。
- “环境声持续存在”，例如雨声、门外砸门声、电话震动、玻璃杯轻响、车辆远去声。
- “台词与口型同步”，但不要机械重复；只在易混淆的画外音、心声、电话音和多人对话中显式说明。
- “对话期间保持眼神交汇、头部微转、轻微肢体反应”，用于承载对话关系。

## 与 Seedance 口径的区别

- Seedance 分镜更强调现有工程合同：10-15 秒组、0.5 秒粒度、组首空间锁定、组尾衔接、强节拍密度和可校验结构。
- HappyHorse 分镜在保留工程合同的前提下，更强调每段文本本身像一段完整的视频生成提示词：场景、主体、运动、音频要在自然语言里更顺、更集中。
- HappyHorse 的镜头运动应更克制稳定，优先自然微推、侧推、过肩、平视中近景和反应镜头；不要默认复杂一镜到底或炫技运动。
- HappyHorse 的音频表达要更具体：谁说、对谁说、语气、是否口型同步、环境声是否持续，都要在容易混淆处写清。

## 不要吸收的内容

- 不要把官方 case 的人物、产品、广告语、品牌、Logo、台词或剧情复制进项目分镜。
- 不要默认使用广告带货、产品特写、口播卖点、字幕卡、角标、小字说明。
- 不要输出 `@图片1`、`@视频1`、`Image 1`、`参考图 1`、`首帧参考` 等控制台占位符。
- 不要沿用 Seedance 默认分镜外观：`**人物**`、`**场景**`、`**道具**`、`组首空间锁定`、`画面风格：`、`【技术参数】`、独立 `--neg` 行。
- 不要把每组写成复杂的一镜到底，除非用户明确要求且剧情天然适合。
- 不要把 10-15 秒短剧组塞入过多空间转换、强动作、长台词和外部事件。
- 不要用抽象情绪词替代可见细节；“压迫感”“治愈感”“野心”必须通过动作、光影、构图、道具和台词呈现。

## 分镜生成前检查

1. 按 `【场景】`、`【主体】`、`【运动】`、`【音频】`、`【画面风格】` 检查每组是否完整。
2. 检查必须保留项：台词、动作阶段、关键道具、反转点和声音设计不得被压缩掉。
3. 检查五维锚点：主体、场景、运动、镜头、声音是否都足够可见、可执行。
4. 检查对话口型：现场开口、画外音、电话音、心声必须分离；完整台词只放 `【音频】`。
5. 检查运动负载：HappyHorse 可以多镜头自然转场，但一组仍应控制在 2-4 个清晰节拍，避免强节拍过载。
6. 检查最终正文：不得出现模型说明、官方示例标签、参考图占位符或非短剧生产说明。

## Reviewer 对应检查

reviewer 应把下列情况视为 `prompt_pollution` 或相关问题：

- 最终分镜出现“HappyHorse 自动”“模型会处理”“参考官方 case”等模型说明。
- 最终分镜出现 `@图片`、`@视频`、`Image 1`、`参考图 1`、`首帧参考` 等生产占位符。
- HappyHorse 目标下继续使用 `**人物**`、`**场景**`、`**道具**`、`组首空间锁定`、`画面风格：`、`【技术参数】`、独立 `--neg` 行，导致输出仍像默认 Seedance 分镜。
- `【画面风格】` 中写入 `16:9`、`9:16`、`11秒`、`11s`、`4K`、`1080p` 等生产界面参数。
- `【画面风格】` 里长串重复负向词超过 8 项，或每组重复大量无差别负向词。
- `【运动】` 中重复写完整台词，导致 `【音频】` 再写一遍台词，形成双重提示。
- 为迎合模型示例而新增广告、产品、品牌、Logo、字幕、口播卖点或原剧本没有的道具文字。
- 画外音、心声、电话音被误写成画面人物现场开口，导致口型错误。
- 同一组塞入过多镜头、动作、外部事件和长台词，导致 HappyHorse 无法稳定完成自然表演。
