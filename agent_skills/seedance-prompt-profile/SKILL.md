# Seedance 2.0 Prompt Profile

本文件整理 Seedance 2.0 官方模板的提示词结构经验，用作 Auto-Storyboard 的内部参考层。

它不是分镜生成模板，不是剧情模板，也不是最终分镜正文的一部分。生成分镜时只借鉴结构和表达习惯，不复制官方模板原文。

## 使用原则

- 每次只按当前剧情类型选择 1-2 个最相关的 profile 风格参考。
- 不要全量加载官方模板库，不要把模板正文塞进每次生成上下文。
- 模板编号只用于内部选择参考，不得写入 `final.txt`。
- `@图片1`、`@视频1`、`@音频1` 是官方控制台占位写法；Auto-Storyboard 由 Web 工程通过 `asset_bindings` 和参考图槽位注入资产，不在最终分镜里手写这些占位符。
- Seedance profile 只优化视频模型理解方式；工程合同仍以 `cut_id`、0.5 秒节拍、片段复杂度预算、`storyboard_index`、`asset_bindings`、reviewer 和 validate gate 为准。

## 短剧优先参考

优先参考短剧、对白、对峙和情绪叙事相关 profile。产品广告、萌宠、MV、视频编辑和轨道补全模板默认不进入短剧主流程。

| 对峙 / 智性博弈 / 短句交锋 | 8, 47 | 短时间段内的问答节拍、反应镜头、压迫关系 |

## Seedance 友好的提示词结构

生成分镜时可吸收以下抽象结构：

1. 总时长明确：每组总时长 10-15 秒，并在标题中写清。
2. 时间段明确：组内用连续时间段承载剧情节拍，可使用 0.5 秒粒度。
3. 参考图职责明确：参考图锁定人物、服装、场景和道具外观；时间段承载剧情动作、台词、反应和转折。
4. 动作链完整：连续动作需要完整表演时间，不和长台词、外部事件或关键道具揭示硬塞。
5. 声音来源明确：现场对白、画外音、电话音、广播音、环境音要区分，必要时写明嘴唇闭合、不做口型。
6. 风格统一：真人短剧、竖屏、4K、自然光影、无字幕、无水印、无广告字卡等尾部约束保持稳定。
7. 过渡方式明确：若是蒙太奇、回忆、片尾意象、屏幕画面或主观想象，必须明说，避免被模型理解为同一现实空间。
8. 不机械展示外观：不要用时间段展示角色长相、场景全貌或普通道具外观，除非外观变化本身推动剧情。

## 不要吸收的内容

- 具体模板编号、模板标题、官方模板说明。
- 官方模板里的广告 slogan、角标、小字、字幕逻辑和商业展示语气。
- 萌宠、MV、变装、产品展示等非短剧默认风格。
- `@图片1`、`@视频1`、`@音频1` 等控制台占位符。
- 不适合短剧主流程的全程一镜到底、无剪辑断点、卡点快切、文字贴片和炫技运镜。
- 为追求视觉花哨牺牲剧情节奏、口型自然度、角色状态稳定和分镜可审核性。

## 生成前选择口径

1. 判断当前组的主要任务：对白推进、对峙压迫、动作追逐、温情收尾、道具揭示或特殊转场。
2. 选择 1-2 个最接近的 profile 类别，只借鉴结构，不复述模板内容。
3. 先满足 Auto-Storyboard 的硬规则：原剧本忠实、组总时长、0.5 秒节拍、复杂度预算、音画分离、cut_id。
4. 再用 Seedance 友好表达润色：动作链完整、声音来源清楚、光影质感统一、负面提示稳定。

## Reviewer 对应检查

reviewer 应把下列情况视为 `prompt_pollution` 或相关问题：

- 最终分镜出现模板编号、官方模板标题或“参考官方模板”等说明。
- 最终分镜出现 `@图片`、`@视频`、`@音频` 占位符。
- 短剧分镜混入广告 slogan、角标、小字、产品卖点字幕、MV 卡点说明或萌宠/变装模板语气。
- 因套用模板导致原剧本台词、动作、人物关系或剧情节拍被改写。
- 因追求一镜到底、炫技运镜或广告展示，压缩台词口型、动作表演或关键情绪转折。
